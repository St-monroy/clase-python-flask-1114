Si estas viendo esto, ya diste un primer paso en programacion: tu aplicacion esta corriendo y Flask pudo mostrar una pagina HTML.

Hoy empezamos simple, pero con una idea poderosa: cada gran programa arranca con una primera linea de codigo. Este puede ser el comienzo de algo enorme.

Primer objetivo cumplido: crear el entorno y levantar el servidor.

1.Que problema concreto resuelve el entorno virtual en un proyecto Python?
R/ Evita conflictos de versiones entre distintos proyectos al aislarlos.

2. Que diferencia hay entre instalar `Flask` globalmente y hacerlo dentro de `.venv`?
Global afecta a toda la computadora; .venv solo a la carpeta del proyecto actual.

3. Por que `requirements.txt` forma parte del proyecto y no de tu maquina personal?
R/ Permite que cualquier persona replique exactamente tus librerías en otro equipo.

4. Cuando ejecutas `python app.py`, que archivo actua como punto de entrada y por que?
R/ Es el archivo que contiene el comando para iniciar el servidor de Flask.

5. Que relacion hay entre la ruta `/`, la funcion `inicio()` y el archivo `templates/index.html`?
R/ La ruta detecta la visita, la función procesa la lógica y el HTML es el resultado visual enviado al usuario.

6. Que evidencia te da la terminal de que el servidor arranco correctamente?
R/ Evidencia terminal: El mensaje que indica una URL local (ej. http://127.0.0.1:5000)

7. Si cambias el HTML y el navegador muestra otra cosa, que te demuestra eso sobre el flujo entre backend y frontend en este proyecto?
R/ Demuestra que el backend sirve el contenido dinámicamente; cada petición lee el archivo actual.
